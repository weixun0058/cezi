# 诸葛神算 - 工作流程

本文档记录了诸葛神算项目的工作流程和开发指南，帮助开发者理解项目结构和功能实现方式。

## 最近解决的问题

### 移动设备适配优化

**问题描述**：
- 网站在移动设备（<=480px宽度）上显示效果不佳
- 字体过大或过小，布局不合理
- 交互元素尺寸不适合触摸操作
- 黄历页面在小屏幕上信息过于拥挤

**解决方案**：
1. 创建专门的移动设备CSS文件
   - 创建`style_mobile.css`作为基础移动端样式
   - 创建`huangli_mobile.css`作为黄历页面的移动端样式
   - 调整字体大小、间距和布局以适应小屏幕

2. 使用媒体查询实现自适应加载
   - 在HTML文件中添加媒体查询条件
   - 针对不同屏幕尺寸加载不同的CSS文件
   ```html
   <link rel="stylesheet" media="screen and (min-width: 481px)" href="{{ url_for('static', filename='css/style.css') }}">
   <link rel="stylesheet" media="screen and (max-width: 480px)" href="{{ url_for('static', filename='css/style_mobile.css') }}">
   ```

3. 优化移动端布局
   - 调整容器宽度为百分比而非固定像素
   - 减小页面元素间距和内边距
   - 优化按钮和输入框尺寸，适合触摸操作
   - 调整导航链接布局，确保易于点击

4. 黄历页面的移动端优化
   - 将双列布局改为单列布局
   - 调整宜忌区域为上下布局而非左右布局
   - 减少九天黄历网格为2列而非3列
   - 优化信息卡片内部间距和字体大小

5. 保持视觉风格一致性
   - 保持与桌面版相同的颜色方案和设计语言
   - 确保所有功能在移动端都可用
   - 优化动画效果，减少在移动设备上的性能消耗

### 黄历页面布局进一步优化

**问题描述**：
- 彭祖百忌卡片位置不合理，只占据页面一半宽度
- 日期信息和节气节日卡片未能上下对齐
- 宜忌卡片需要贯通整个页面宽度，宜忌各占左右一半

**解决方案**：
1. 重新设计黄历容器内部布局结构
   - 采用三行布局：顶部信息行、中部宜忌行、底部神煞和彭祖百忌行
   - 顶部行使用网格布局，将日期信息和节气节日卡片并排放置
   - 中部行宜忌卡片贯通整个页面宽度
   - 底部行使用纵向布局，神煞信息在上，彭祖百忌在下，均贯通整个页面宽度

2. 调整CSS样式
   - 更新了`.huangli-content`的布局方式，使用纵向flex布局
   - 为顶部行创建了网格布局，确保两个卡片等宽
   - 中部行和底部行使用100%宽度，确保内容贯通整个页面

3. 保持宜忌区域的左右布局
   - 维持宜忌区域内部的flex布局，确保宜和忌各占50%宽度
   - 保持标签容器的样式不变，确保内容显示一致

4. 优化响应式布局
   - 在小屏幕设备上调整顶部行为单列布局
   - 确保所有内容在各种屏幕尺寸下都能清晰展示

### 黄历页面内部布局优化

**问题描述**：
- 黄历容器内部布局不合理，信息展示缺乏层次感
- 各部分内容之间的关系不够清晰，影响用户体验
- 移动设备上的显示效果不佳

**解决方案**：
1. 重新设计黄历容器内部布局
   - 采用左右两栏加底部区域的网格布局方式
   - 左侧栏放置日期信息和节气节日
   - 右侧栏放置宜忌和彭祖百忌
   - 底部区域放置神煞信息

2. 统一信息卡片样式
   - 为所有内容区域添加统一的卡片样式
   - 添加卡片标题，使各部分内容更加清晰
   - 优化内部间距和边框，提升视觉美感

3. 改进响应式布局
   - 在小屏幕设备上自动调整为单列布局
   - 优化宜忌区域在移动设备上的显示方式
   - 确保所有内容在各种屏幕尺寸下都能清晰展示

4. 保留原有功能的同时提升用户体验
   - 保持所有数据展示的完整性
   - 优化信息的视觉层次，突出重要内容
   - 确保样式与整体页面风格保持一致

### 九天黄历数据加载问题

**问题描述**：
- 前端无法加载九天黄历数据，显示"获取九天黄历数据出错"
- API请求失败，返回连接错误
- DOM元素ID不匹配，导致数据无法正确显示
- CSS样式问题导致宜忌内容布局错误

**解决方案**：
1. 修复API端口问题
   - 应用实际运行在80端口而非5000端口
   - 更新前端代码中的API请求URL，确保正确访问

2. 修复DOM元素ID不匹配问题
   - HTML中使用的是`weekHuangliContainer`，而JavaScript中错误使用了`week-huangli`
   - 统一使用`weekHuangliContainer`作为九天黄历容器的ID

3. 修复CSS样式问题
   - 将`.suitable-unsuitable`的`display`属性从`contents`改为`flex`
   - 确保宜忌内容正确显示并保持对称布局

4. 添加详细调试日志
   - 在页面加载完成时检查关键DOM元素是否存在
   - 在数据处理过程中添加详细日志，便于问题排查
   - 记录API响应状态和数据处理过程

### 时辰显示问题修复

**问题描述**：
- 日期信息中的时辰显示与实际不符
- 后端返回的干支时辰信息未被正确显示
- 前端代码中只显示了时辰名称部分，忽略了完整的干支信息

**解决方案**：
1. 修复前端JavaScript中的时辰显示逻辑
   - 确保完整显示后端返回的`gan_zhi_hour`信息
   - 移除了将时辰信息拆分的代码，保留完整的干支时辰
   - 当后端未返回时辰信息时，使用前端计算的当前时辰作为备用

2. 确保时辰信息的实时更新
   - 保留了每分钟更新时辰的功能
   - 确保时辰信息与当前实际时间同步

### 九天黄历卡片布局优化

**问题描述**：
- 九天黄历卡片中的宜忌内容上下排列，不够对称美观
- 神煞信息显示格式不合理，所有信息混在一起
- 需要调整为冲煞单独一行，喜神、福神、财神在第二行

**解决方案**：
1. 重构九天黄历卡片中的宜忌布局

   - 为宜忌各自分配48%的宽度，确保视觉平衡
   - 保留了内容限制为两行的功能，避免过长内容影响布局

2. 优化神煞信息的显示格式
   - 将冲煞信息单独放在第一行
   - 将喜神、福神、财神信息放在第二行，用分隔符连接
   - 使用HTML格式化而非纯文本，支持换行显示
   - 调整了CSS样式，移除了单行省略的限制，确保信息完整显示

3. 改进样式和响应式设计
   - 优化了文本对齐和间距
   - 确保在各种屏幕尺寸下都能正确显示
   - 保持与整体页面风格的一致性

### 论命功能代码优化

**问题描述**：
- 论命功能中存在冗余的stream_analysis函数，与analyze_bazi函数功能重复
- 使用模拟数据作为API调用失败的备选方案，不符合真实性要求
- 代码结构不够清晰，错误处理逻辑分散

**解决方案**：
1. 删除冗余的stream_analysis函数
   - 将stream_analysis函数的流式输出功能合并到analyze_bazi函数中
   - 确保analyze_bazi函数能够正确处理所有参数并生成流式输出
   - 保持API接口的一致性，不影响前端调用

2. 移除模拟数据功能
   - 删除API调用失败后使用模拟数据的逻辑
   - 当API调用失败时，直接返回明确的错误信息
   - 确保所有分析结果都来自真实的AI模型调用

3. 优化错误处理逻辑
   - 统一错误处理方式，确保所有错误都能被正确捕获和报告
   - 添加详细的错误日志，便于调试和问题定位
   - 确保错误信息能够以流式方式返回给前端，提供更好的用户体验

4. 保持代码结构清晰
   - 确保函数职责明确，避免功能重复
   - 优化代码注释，提高可读性
   - 保持与项目其他部分的代码风格一致

3. 重新设计论命页面

   - 添加分析按钮和重置按钮
   - 添加结果展示区域，支持流式显示分析结果

4. 创建专用CSS样式
   - 创建`lunming.css`文件，为论命页面添加专用样式
   - 优化表单布局和输入控件样式

5. 实现前端JavaScript逻辑

## 功能模块

### 黄历模块 (huangli.py)

黄历模块负责生成和管理传统黄历数据，主要功能包括：

1. **数据生成**：
   - `_generate_huangli_data(date)`: 根据日期生成黄历数据
   - 使用lunar-python库获取农历信息、干支、宜忌等数据

2. **数据缓存**：
   - `_init_db()`: 初始化数据库表结构
   - `_save_huangli_to_db(date, data)`: 将黄历数据保存到数据库
   - `_get_huangli_from_db(date)`: 从数据库获取缓存的黄历数据

3. **API接口**：
   - `get_daily_huangli(date)`: 获取指定日期的黄历数据
   - `get_week_huangli()`: 获取九天（前2天、今天和未来6天）的黄历数据

### 算事模块 (suanshi.py)

算事模块负责处理用户的算卦请求，主要功能包括：

1. **数据处理**：
   - 接收用户输入的姓名、性别、出生日期和问题
   - 根据输入信息计算卦象

2. **结果生成**：
   - 根据卦象生成解签结果
   - 提供事业、财运、情感等方面的解读

### 论命模块 (lunming.py)

论命模块负责八字命理分析，目前处于开发中，计划功能包括：

1. **八字计算**：
   - 根据用户出生年月日时生成八字
   - 计算命盘信息

2. **命理解读**：
   - 提供命理分析和人生指导
   - 生成详细的命理报告

## 前端开发

### 页面结构

所有页面遵循统一的布局结构：

1. **头部**：标题和导航链接
2. **诗词部分**：展示与功能相关的古诗词
3. **主要内容区**：功能特定的内容
4. **卷轴容器**：使用古风卷轴样式包裹内容

### 响应式设计原则

1. **移动优先设计**：
   - 优先考虑移动设备的用户体验
   - 使用相对单位（如百分比、em、rem）而非固定像素
   - 使用媒体查询针对不同屏幕尺寸优化布局

2. **断点设置**：
   - 主要断点：480px（移动设备与桌面设备的分界点）
   - 使用媒体查询在不同断点加载不同CSS文件

3. **布局调整策略**：
   - 桌面版：多列布局，充分利用宽屏空间
   - 移动版：单列或双列布局，确保内容清晰可读
   - 导航菜单在移动版上适当调整大小和间距

4. **触摸友好设计**：
   - 确保所有可点击元素尺寸足够大（至少44x44像素）
   - 增加元素间距，避免误触
   - 优化表单元素，适合移动设备输入

### 样式指南

1. **颜色方案**：
   - 主色调：#8b4513（棕色）
   - 背景色：#fff8dc（米色）
   - 强调色：#d4a76a（金色）

2. **字体**：
   - 标题：使用古风字体
   - 正文：使用清晰易读的字体

3. **组件样式**：
   - 按钮：古风按钮样式
   - 卡片：带有轻微阴影和边框的卡片
   - 标签：圆角标签样式

## 流式输出实现流程

### 服务器端实现
1. `app.py` 中的 `stream_bazi_analysis` 函数接收用户输入数据
2. 调用 `lunming.py` 中的 `analyze_bazi` 方法生成分析结果
3. 将每个字符作为JSON对象返回给前端
4. 使用 `yield` 实现流式响应，确保响应头设置为 `text/event-stream`
5. 完成分析后，发送 `{"done": true}` 信号表示传输完成

### 前端实现
1. 前端使用 `EventSource` API 连接服务器端的流式输出接口
2. 接收服务器发送的每个字符，添加到文本缓冲区
3. 通过缓冲区实现智能文本解析：
   - 检测段落结束（空行）
   - 识别并格式化 Markdown 风格标题
   - 识别并格式化数字标题
   - 识别并格式化列表项
4. 实时更新DOM，显示分析结果
5. 接收完成信号后，关闭连接并隐藏加载动画

## 技术要点
1. 流式响应技术：
   - 后端使用 `yield` 和 `Response(stream_with_context())` 实现流式输出
   - 前端使用 `EventSource` API 实现持续接收
2. 文本解析技术：
   - 使用正则表达式检测不同类型的文本格式
   - 基于行的解析替代基于字符的解析
3. DOM操作优化：
   - 使用缓冲区减少DOM操作次数
   - 针对不同类型内容创建不同样式的DOM元素
