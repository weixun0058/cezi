# 诸葛神算 - 工作流程

本文档记录了诸葛神算项目的工作流程和开发指南，帮助开发者理解项目结构和功能实现方式。

## 最近解决的问题

### 黄历页面布局进一步优化

**问题描述**：
- 彭祖百忌卡片位置不合理，只占据页面一半宽度
- 日期信息和节气节日卡片未能上下对齐
- 宜忌卡片需要贯通整个页面宽度，宜忌各占左右一半

**解决方案**：
1. 重新设计黄历容器内部布局结构
   - 采用三行布局：顶部信息行、中部宜忌行、底部神煞和彭祖百忌行
   - 顶部行使用网格布局，将日期信息和节气节日卡片并排放置
   - 中部行宜忌卡片贯通整个页面宽度
   - 底部行使用纵向布局，神煞信息在上，彭祖百忌在下，均贯通整个页面宽度

2. 调整CSS样式
   - 更新了`.huangli-content`的布局方式，使用纵向flex布局
   - 为顶部行创建了网格布局，确保两个卡片等宽
   - 中部行和底部行使用100%宽度，确保内容贯通整个页面

3. 保持宜忌区域的左右布局
   - 维持宜忌区域内部的flex布局，确保宜和忌各占50%宽度
   - 保持标签容器的样式不变，确保内容显示一致

4. 优化响应式布局
   - 在小屏幕设备上调整顶部行为单列布局
   - 确保所有内容在各种屏幕尺寸下都能清晰展示

### 黄历页面内部布局优化

**问题描述**：
- 黄历容器内部布局不合理，信息展示缺乏层次感
- 各部分内容之间的关系不够清晰，影响用户体验
- 移动设备上的显示效果不佳

**解决方案**：
1. 重新设计黄历容器内部布局
   - 采用左右两栏加底部区域的网格布局方式
   - 左侧栏放置日期信息和节气节日
   - 右侧栏放置宜忌和彭祖百忌
   - 底部区域放置神煞信息

2. 统一信息卡片样式
   - 为所有内容区域添加统一的卡片样式
   - 添加卡片标题，使各部分内容更加清晰
   - 优化内部间距和边框，提升视觉美感

3. 改进响应式布局
   - 在小屏幕设备上自动调整为单列布局
   - 优化宜忌区域在移动设备上的显示方式
   - 确保所有内容在各种屏幕尺寸下都能清晰展示

4. 保留原有功能的同时提升用户体验
   - 保持所有数据展示的完整性
   - 优化信息的视觉层次，突出重要内容
   - 确保样式与整体页面风格保持一致

### 九天黄历数据加载问题

**问题描述**：
- 前端无法加载九天黄历数据，显示"获取九天黄历数据出错"
- API请求失败，返回连接错误
- DOM元素ID不匹配，导致数据无法正确显示
- CSS样式问题导致宜忌内容布局错误

**解决方案**：
1. 修复API端口问题
   - 应用实际运行在80端口而非5000端口
   - 更新前端代码中的API请求URL，确保正确访问

2. 修复DOM元素ID不匹配问题
   - HTML中使用的是`weekHuangliContainer`，而JavaScript中错误使用了`week-huangli`
   - 统一使用`weekHuangliContainer`作为九天黄历容器的ID

3. 修复CSS样式问题
   - 将`.suitable-unsuitable`的`display`属性从`contents`改为`flex`
   - 确保宜忌内容正确显示并保持对称布局

4. 添加详细调试日志
   - 在页面加载完成时检查关键DOM元素是否存在
   - 在数据处理过程中添加详细日志，便于问题排查
   - 记录API响应状态和数据处理过程

### 时辰显示问题修复

**问题描述**：
- 日期信息中的时辰显示与实际不符
- 后端返回的干支时辰信息未被正确显示
- 前端代码中只显示了时辰名称部分，忽略了完整的干支信息

**解决方案**：
1. 修复前端JavaScript中的时辰显示逻辑
   - 确保完整显示后端返回的`gan_zhi_hour`信息
   - 移除了将时辰信息拆分的代码，保留完整的干支时辰
   - 当后端未返回时辰信息时，使用前端计算的当前时辰作为备用

2. 确保时辰信息的实时更新
   - 保留了每分钟更新时辰的功能
   - 确保时辰信息与当前实际时间同步

### 九天黄历卡片布局优化

**问题描述**：
- 九天黄历卡片中的宜忌内容上下排列，不够对称美观
- 神煞信息显示格式不合理，所有信息混在一起
- 需要调整为冲煞单独一行，喜神、福神、财神在第二行

**解决方案**：
1. 重构九天黄历卡片中的宜忌布局
   - 创建了新的容器结构，使宜忌内容左右对称排布
   - 为宜忌各自分配48%的宽度，确保视觉平衡
   - 保留了内容限制为两行的功能，避免过长内容影响布局

2. 优化神煞信息的显示格式
   - 将冲煞信息单独放在第一行
   - 将喜神、福神、财神信息放在第二行，用分隔符连接
   - 使用HTML格式化而非纯文本，支持换行显示
   - 调整了CSS样式，移除了单行省略的限制，确保信息完整显示

3. 改进样式和响应式设计
   - 优化了文本对齐和间距
   - 确保在各种屏幕尺寸下都能正确显示
   - 保持与整体页面风格的一致性

1. 应用入口：`app.py`
2. 初始化数据库：首次运行时会自动创建数据库
3. 启动Web服务器：默认在5000端口运行

## 功能模块

### 黄历模块 (huangli.py)

黄历模块负责生成和管理传统黄历数据，主要功能包括：

1. **数据生成**：
   - `_generate_huangli_data(date)`: 根据日期生成黄历数据
   - 使用lunar-python库获取农历信息、干支、宜忌等数据

2. **数据缓存**：
   - `_init_db()`: 初始化数据库表结构
   - `_save_huangli_to_db(date, data)`: 将黄历数据保存到数据库
   - `_get_huangli_from_db(date)`: 从数据库获取缓存的黄历数据

3. **API接口**：
   - `get_daily_huangli(date)`: 获取指定日期的黄历数据
   - `get_week_huangli()`: 获取九天（前2天、今天和未来6天）的黄历数据

### 算事模块 (suanshi.py)

算事模块负责处理用户的算卦请求，主要功能包括：

1. **数据处理**：
   - 接收用户输入的姓名、性别、出生日期和问题
   - 根据输入信息计算卦象

2. **结果生成**：
   - 根据卦象生成解签结果
   - 提供事业、财运、情感等方面的解读

### 论命模块 (lunming.py)

论命模块负责八字命理分析，目前处于开发中，计划功能包括：

1. **八字计算**：
   - 根据用户出生年月日时生成八字
   - 计算命盘信息

2. **命理解读**：
   - 提供命理分析和人生指导
   - 生成详细的命理报告
