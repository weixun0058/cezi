# 英文版黄历修订方案

> 制定时间：2026-07-12
> 状态：待实施
> 依据：用户意见 + 后端现状调研 + 产品规格文档

## 一、背景

英文版黄历（Daily Almanac，路由 `/daily-almanac`）当前前端展示不完整：
- 后端 `huangli_english.py` 已翻译并返回多个字段，但前端 `daily_almanac.js` 未渲染
- 后端 `/api/en/week-almanac` 已就绪，但前端未调用
- 部分内容（彭祖百忌）不适合英文用户，需移除
- 专业术数概念（二十八宿/建除十二神/胎神占方）中英文版都不需要

本方案明确"补什么、删什么、不做什么"，作为实施的追溯依据。

## 二、设计原则

1. **前端缺失补前端**：后端已返回的数据，前端补渲染即可
2. **彭祖百忌英文版移除**：中文谚语体翻译后失去韵律且难以理解，前后端都移除
3. **专业术数概念不要**：二十八宿/建除十二神/胎神占方非专业人士看不懂，中英文版都不做
4. **实时时钟不要**：十二时辰对英文用户意义有限，增加复杂度
5. **干支用阴阳五行+生肖方案**：后端已实现，如 "Yang Fire Horse Year"
6. **中文版维持现状**：本次只改英文版，不影响中文版黄历

## 三、改动清单

### 3.1 英文版前端补充（后端已返回数据，前端补渲染）

| 序号 | 补充项 | 后端字段 | 当前状态 | 改动内容 |
|---|---|---|---|---|
| F1 | 干支展示（年月日时） | `gan_zhi.year/month/day/hour` | 后端已翻译为 "Yang Fire Horse Year" 格式 | 前端新增干支卡片，渲染 4 个字段 |
| F2 | 前后节气及天数 | `solar_term.previous/next` | 前端只展示 current | 前端补充 previous/next 节气名及距今天数 |
| F3 | 节日列表 | `festivals[]` | 后端已返回，前端未展示 | 前端新增节日展示区 |
| F4 | 特殊规则 | `special_indications[]` | 英文版独有字段但未用 | 前端渲染"馀事勿取"等特殊规则提示 |
| F5 | 九天黄历网格 | `/api/en/week-almanac` | 后端 API 已就绪，前端未调用 | 前端新增可折叠九天网格（前2天+今天+后6天） |
| F6 | 前/今/后日期导航按钮 | 无（前端交互） | 仅靠日期选择器 | 新增 3 个导航按钮，快捷切换日期 |

### 3.2 英文版前后端移除

| 序号 | 移除项 | 范围 | 改动内容 |
|---|---|---|---|
| R1 | 彭祖百忌 | 前端 + 后端 | 前端移除彭祖百忌卡片；后端 `huangli_english.py` 的 `translate_daily` 移除 `peng_zu_taboos` 字段 |

### 3.3 明确不做的内容

| 序号 | 内容 | 不做原因 |
|---|---|---|
| N1 | 实时时钟+十二时辰 | 对英文用户意义有限，增加复杂度 |
| N2 | 彭祖百忌英文解释 API | 彭祖百忌整体移除，无需解释 API |
| N3 | 二十八宿 | 非专业人士看不懂，中英文版都不做 |
| N4 | 建除十二神 | 同上 |
| N5 | 胎神占方 | 同上 |
| N6 | 宜忌细粒度还原 | 英文版 39 个粗粒度类别更适合英文用户，保持现状 |

### 3.4 中文版

维持现状，不做任何改动。中文版彭祖百忌保留，二十八宿等本就没有。

## 四、干支英文展示格式（已确认）

后端 `huangli_english.py` 的 `translate_gan_zhi` 函数已实现，数据源为 `huangli_terms_en.json` 的 `sexagenary` 命名空间（60 条完整翻译）。

格式：**阴阳 + 五行 + 生肖 + 后缀**

示例：
- 甲子年 → Yang Wood Rat Year
- 丙午月 → Yang Fire Horse Month
- 辛卯日 → Yin Metal Rabbit Day
- 戊戌时 → Yang Earth Dog Hour

后端已就绪，前端只需渲染 `gan_zhi` 字段。

## 五、技术实现要点

### 5.1 前端文件

- 模板：`frontend/templates/en/daily_almanac.html`
- JS：`frontend/static/js/daily_almanac.js`
- CSS：`frontend/static/css/wise_oracle.css`（`.en-almanac*` 区段）

### 5.2 后端文件

- API 路由：`zhugeshensuan/blueprints/huangli_en_api.py`
- 翻译服务：`zhugeshensuan/huangli_english.py`（移除 `peng_zu_taboos`）

### 5.3 九天黄历网格实现

- 调用 `/api/en/week-almanac?scenario=`
- 可折叠容器 + 九天网格卡片
- 场景筛选按钮组（General/Wedding/Moving/Business Opening/Travel/Signing/Haircut），切换时重新调用 API
- 参考中文版 `huangli.html` 的九天网格交互

### 5.4 前/今/后导航按钮

- "◀ Previous Day" / "Today" / "Next Day ▶" 三个按钮
- 点击后更新日期选择器并重新调用 `/api/en/daily-almanac`
- 参考中文版 `huangli.js` 的导航逻辑
