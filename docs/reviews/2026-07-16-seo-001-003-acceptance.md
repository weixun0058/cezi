## 2. 已实施范围

### SEO-001：可信 URL、canonical 与 hreflang

- 新增唯一公开 URL 配置 `SITE_BASE_URL`，生产环境只接受公开 HTTPS origin；拒绝空值、localhost、IP、`.example`、用户信息、path、query 和 fragment。
- canonical、hreflang、Open Graph URL、JSON-LD URL、sitemap 和 robots 均从配置生成，不读取请求 `Host` 或转发 host。
- 新增集中式不可变页面注册表，统一管理英文、简体和繁体页面的 title、description、canonical、robots、语言与对应关系。
- 四组核心页面完整互返 `en`、`zh-Hans`、`zh-Hant`、`x-default`；没有中文等价内容的信任页不伪造 hreflang。
- `.env.example`、真实本地 `.env`、生产 Compose、CI 和部署指南均已补入 `SITE_BASE_URL=https://getwiseoracle.com`。真实 `.env` 不纳入 Git，也未在验收输出中暴露其他变量。

### SEO-002：sitemap 与 robots

- 新增 `/sitemap.xml`，精确包含批准的 9 个英文及繁体公开 URL，不从 Flask 路由表自动推导。
- 新增 `/robots.txt`，允许公开页面抓取，提示不抓取 API 和健康探针，并唯一指向正式 sitemap。
- 未发布文章详情改为 404；`/articles` Coming soon 页面保持 `noindex,follow`。

### SEO-003：页面元数据、Open Graph 与结构化数据

- 英文和中文模板统一输出唯一 title、description、canonical、robots 与 Open Graph 元数据。
- `og:locale` 固定映射为 `en_US`、`zh_CN`、`zh_TW`。
- 首页输出 `WebSite` 与 `WebPage`，其他批准页面输出保守的 `WebPage`；不虚构 Article、评分、评论、价格、资质或保证结果。
- 暂未加入 `og:image`：当前没有完成版权、尺寸和分享预览验收的 1200×630 品牌图，避免引用不存在或不合格的资产。
- 中文首页恢复 480px 以下的移动样式表加载；这是浏览器验收发现并修复的既有模板缺口。

## 8. 回滚方式

本批次没有数据库迁移。若部署后出现模板渲染、canonical、sitemap 或 robots 错误：

1. 回滚到部署前应用镜像或提交，不删除 `SITE_BASE_URL` 来绕过生产校验。
2. 清理 CDN 中错误的 HTML、sitemap 与 robots 缓存。
3. 复测首页、三个英文工具页、繁体页面、sitemap、robots、healthz 和 readyz。
4. canonical 指向错误域名时按高优先级发布事故处理，并记录错误 URL、故障窗口、回滚提交和恢复时间。
