以下是 Flask 项目部署到服务器（以 **Ubuntu + Nginx + Gunicorn** 为例）的详细步骤：

---

### **一、准备工作**
1. **服务器环境**  
   - 确保已购买云服务器（如腾讯云轻量应用服务器）。  
   - 通过 SSH 登录服务器：  
     ```bash
     ssh root@服务器IP
     ```

2. **更新系统及安装基础工具**  
   ```bash
   apt update && apt upgrade -y
   apt install -y python3 python3-pip python3-venv nginx git
   ```

3. **上传项目代码到服务器**  
   - **方法 1：使用 `scp` 上传本地代码**  
     ```bash
     scp -r /本地/项目路径 root@服务器IP:/home/项目目录
     ```

   - **方法 2：通过 Git 克隆远程仓库**  
     ```bash
     git clone https://github.com/你的用户名/项目仓库.git
     ```

---

### **二、配置 Python 虚拟环境**
1. **创建并激活虚拟环境**  
   ```bash
   cd /home/项目目录
   python3 -m venv venv
   source venv/bin/activate  # 激活虚拟环境（提示符前显示 (venv)）
   ```

2. **安装项目依赖**  
   ```bash
   pip install -r requirements.txt  # 确保项目根目录有 requirements.txt
   pip install gunicorn            # 安装 Gunicorn（WSGI 服务器）
   ```

---

### **三、配置 Gunicorn**
1. **测试启动 Flask 应用**  
   ```bash
   gunicorn -w 4 -b 0.0.0.0:5000 你的应用入口文件:应用实例名
   ```
   - 示例：入口文件为 `app.py`，应用实例为 `app`  
     ```bash
     gunicorn -w 4 -b 0.0.0.0:5000 app:app
     ```
   - 浏览器访问 `http://服务器IP:5000` 确认是否正常响应。

2. **创建 Systemd 服务（后台运行）**  
   - 新建服务文件：  
     ```bash
     nano /etc/systemd/system/flaskapp.service
     ```
   - 写入以下内容（根据实际情况修改路径）：  
     ```ini
     [Unit]
     Description=Gunicorn instance for Flask App
     After=network.target

     [Service]
     User=root
     Group=www-data
     WorkingDirectory=/home/项目目录
     Environment="PATH=/home/项目目录/venv/bin"
     ExecStart=/home/项目目录/venv/bin/gunicorn -w 4 -b unix:/tmp/flaskapp.sock app:app

     [Install]
     WantedBy=multi-user.target
     ```
   - 保存后启用并启动服务：  
     ```bash
     systemctl daemon-reload
     systemctl start flaskapp
     systemctl enable flaskapp  # 设置开机自启
     ```

---

### **四、配置 Nginx 反向代理**
1. **创建 Nginx 配置文件**  
   ```bash
   nano /etc/nginx/sites-available/flaskapp
   ```
   - 写入以下配置（替换 `server_name` 为你的域名或IP）：  
     ```nginx
     server {
         listen 80;
         server_name 你的域名或IP;

         location / {
             proxy_pass http://127.0.0.1:5000;  # 你的Flask应用地址
             
             # SSE支持配置
             proxy_buffering off;
             proxy_cache off;
             proxy_set_header Connection '';
             proxy_http_version 1.1;
             chunked_transfer_encoding off;
             
             # 超时设置
             proxy_read_timeout 300s;
             proxy_connect_timeout 300s;
             proxy_send_timeout 300s;
             
             # 代理头信息
             proxy_set_header Host $host;
             proxy_set_header X-Real-IP $remote_addr;
             proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
             proxy_set_header X-Forwarded-Proto $scheme;
         }

         # 若需托管静态文件（CSS/JS/图片）
         location /static {
             alias /home/项目目录/static;
             expires 30d;
         }
     }
     ```

2. **启用配置并重启 Nginx**  
   ```bash
   ln -s /etc/nginx/sites-available/flaskapp /etc/nginx/sites-enabled/
   nginx -t           # 检查配置语法
   systemctl restart nginx
   ```

---

### **五、配置防火墙**
1. **开放 HTTP/HTTPS 端口**  
   ```bash
   ufw allow 80/tcp
   ufw allow 443/tcp
   ufw reload
   ```

2. **关闭调试模式（关键！）**  
   - 在 Flask 代码中设置：  
     ```python
     if __name__ == "__main__":
         app.run(debug=False)  # 生产环境必须禁用 debug
     ```

---

### **六、验证部署**
1. **访问网站**  
   浏览器输入 `http://你的域名或IP`，查看是否正常加载。

2. **查看日志排查问题**  
   - Gunicorn 日志：  
     ```bash
     journalctl -u flaskapp
     ```
   - Nginx 日志：  
     ```bash
     tail -f /var/log/nginx/error.log
     ```

---

### **七、进阶配置（可选）**
1. **HTTPS 证书（Let's Encrypt）**  
   ```bash
   apt install certbot python3-certbot-nginx
   certbot --nginx -d 你的域名
   ```

2. **进程守护工具**  
   使用 `supervisor` 替代 Systemd 管理进程（适合复杂场景）。

---

### **常见问题**
- **502 Bad Gateway**：检查 Gunicorn 是否运行，`/tmp/flaskapp.sock` 权限是否正确。  
- **静态文件无法加载**：确认 Nginx 配置中的 `alias` 路径是否正确。  
- **依赖缺失**：确保虚拟环境中安装了所有 `requirements.txt` 中的包。

---

按此流程操作，你的 Flask 应用即可稳定运行在服务器上！
