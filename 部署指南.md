# 部署指南

## 环境要求

- Python 3.13
- Linux 容器或支持 WSGI 的 Linux 主机
- 可写的 `/app/instance` 持久化目录

生产环境不要使用 Flask 内置服务器，也不要启用 `APP_DEBUG`。

## Docker

```bash
docker build -t zhugeshensuan:4.1 .
docker run -d --name zhugeshensuan \
  -p 8000:8000 \
  --env-file .env \
  -v zhugeshensuan-instance:/app/instance \
  zhugeshensuan:4.1
```

验证：

```bash
curl --fail http://127.0.0.1:8000/healthz
```

## 直接使用 Gunicorn

```bash
python -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
gunicorn --config gunicorn.conf.py app:app
```

## 必要配置

- `SECRET_KEY`：随机长字符串。
- `AI_API_KEY`：模型供应商重新生成的密钥。
- `AI_BASE_URL`、`AI_MODEL`：OpenAI 兼容端点与模型名。
- `AI_TIMEOUT_SECONDS`：模型请求超时。
- `HUANGLI_DB_PATH`：默认 `instance/huangli.db`。
- `MAX_CONTENT_LENGTH`：默认 1 MiB。

密钥必须由部署平台注入，不得写入镜像、Nginx 配置或仓库。

## Nginx

`flaskapp` 提供反向代理示例。SSE 路径必须关闭代理缓冲并使用足够长的读取超时。应用默认不开放跨域；如确需跨域，应显式配置可信来源，不得同时使用通配来源和凭据。

## 数据与升级

`instance/` 需要持久化和定期备份。黄历缓存带算法版本，版本变化时按日期重建，不需要删除整库。`database/kanxi_dict.db` 和 Excel/JSON 数据是只读基础数据。
